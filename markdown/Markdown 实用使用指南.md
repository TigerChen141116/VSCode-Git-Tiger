# Markdown 实用使用指南

Markdown 是一种轻量级标记语言，语法简洁、易上手，无需复杂操作即可快速排版，广泛应用于笔记、文档、博客、GitHub 等场景，核心优势是“所见即所得”的极简编辑体验，以下是完整实用指南。

## 一、基础语法（必学）

### 1. 标题（分级排版）

用 `#` 符号表示标题，# 数量对应标题级别（1-6级），# 后需加一个空格，建议一级标题只出现1个（文档标题）。

语法：

```markdown
# 一级标题（最大，常用作文档标题）
## 二级标题（常用作章节标题）
### 三级标题（常用作小节标题）
#### 四级标题
##### 五级标题
###### 六级标题（最小）
```

效果：对应分级标题，字体大小逐级递减，自带段落间距。

### 2. 段落与换行

- 段落：直接输入文本，段落之间空一行即可分隔（无需按回车多次）。

- 换行：在一行末尾加 2 个及以上空格，再按回车（或直接使用 `<br/>` 标签，适合强制换行）。

示例：

```markdown
这是第一段文本，末尾加两个空格  
这是同一段的换行内容。

这是第二段文本（与上一段空一行）。
```

### 3. 文本样式（加粗、斜体、删除线）

三种常用文本样式，可单独使用或组合使用（如加粗+斜体）。

```markdown
*斜体文本*（用1个星号包裹）
**加粗文本**（用2个星号包裹）
***加粗斜体文本***（用3个星号包裹）
~~删除线文本~~（用2个波浪线包裹）
```

效果：*斜体文本*、**加粗文本**、***加粗斜体文本***、~~删除线文本~~。

### 4. 列表（有序/无序）

#### （1）无序列表

用 `*`、`-`、`+` 任意一种作为列表项标记，标记后加1个空格，可嵌套。

```markdown
- 无序列表项1
- 无序列表项2
  - 嵌套无序列表项（前面加2个空格）
  - 嵌套无序列表项
+ 无序列表项3（用+也可）
```

#### （2）有序列表

用数字 + `.` 作为标记，标记后加1个空格，序号会自动递增，可嵌套。

```markdown
1. 有序列表项1
2. 有序列表项2
   1. 嵌套有序列表项（前面加2个空格）
   2. 嵌套有序列表项
3. 有序列表项3（序号自动衔接）
```

### 5. 引用

用 `>` 表示引用，> 后加1个空格，可嵌套（多一层 >），适合引用他人话语、文献片段。

```markdown
> 这是一级引用文本
> > 这是嵌套的二级引用文本
> 引用结束后，直接换行即可退出引用模式
```

效果：

> 这是一级引用文本
> 
> 这是嵌套的二级引用文本

引用结束后，直接换行即可退出引用模式

### 6. 代码（行内/块级）

用于插入代码片段，区分行内代码（短片段）和块级代码（多行长代码），支持语法高亮。

#### （1）行内代码

用 1 个反引号 ``` 包裹代码片段，适合嵌入句子中。

```markdown
在 Python 中，`print("Hello World")` 是最基础的输出语句。
```

效果：在 Python 中，`print("Hello World")` 是最基础的输出语句。

#### （2）块级代码

用 3 个反引号 ````` 包裹代码，开头的反引号后可加编程语言名称（如 python、java），实现语法高亮，适合多行长代码。

```markdown
```python
# 这是Python代码示例
def hello():
    print("Hello, Markdown!")
hello()
```

```
效果（带语法高亮）：

```python
# 这是Python代码示例
def hello():
    print("Hello, Markdown!")
hello()
```

## 二、进阶语法（常用）

### 1. 链接

两种写法：行内链接（直接显示）、参考链接（适合多次引用同一链接）。

```markdown
# 行内链接（格式：[链接文本](链接地址 "可选备注，鼠标悬停显示")）
[百度](https://www.baidu.com "百度搜索")

# 参考链接（格式：[链接文本][参考标识]，末尾统一定义参考标识）
[GitHub][1]
[Markdown 官方文档][2]

[1]: https://github.com "GitHub官网"
[2]: https://daringfireball.net/projects/markdown/ "Markdown官方指南"
```

效果：[百度](https://www.baidu.com)、[GitHub](https://github.com)。

### 2. 图片

语法与链接类似，只是前面多一个 `!`，格式：`![图片alt文本](图片地址 "可选备注")`。

- alt文本：图片无法加载时显示的文字，提升可读性和兼容性。

- 图片地址：可是本地路径（如 `./images/1.jpg`）或网络图片链接。

```markdown
![Markdown图标](https://example.com/markdown.png "Markdown图标")
![本地图片](./images/photo.jpg "本地图片示例")
```

### 3. 表格

用 `|` 分隔列，`-` 分隔表头和表体，`:` 可设置对齐方式（左对齐、居中、右对齐）。

```markdown
# 基础表格（无对齐）
| 表头1 | 表头2 | 表头3 |
| --- | --- | --- |
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |

# 带对齐的表格（: 位置决定对齐方式）
| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

效果：

| 左对齐 | 居中对齐 | 右对齐 |
| --- | ---- | --- |
| 内容1 | 内容2  | 内容3 |
| 内容4 | 内容5  | 内容6 |

### 4. 分割线

用 3 个及以上 `-`、`*`、`_` 表示，单独占一行，前后建议空一行，避免与标题混淆。

```markdown
---
***
___
```

效果（三种写法一致）：

---

## 三、实用技巧（提升效率）

1. 快捷键（以 VS Code 为例）：
   
   - 加粗：Ctrl + B
   
   - 斜体：Ctrl + I
   
   - 插入链接：Ctrl + K + L
   
   - 插入图片：Ctrl + K + I
   
   - 插入代码块：Ctrl + Shift + ```

2. 兼容问题：不同平台（如 GitHub、Notion、Typora）的 Markdown 渲染略有差异，优先使用基础语法，避免复杂嵌套。

3. 注释：Markdown 本身不支持注释，可使用 `<!-- 注释内容 -->`（仅编辑界面可见，渲染后不显示）。

4. 空行：连续空行可通过 `&nbsp;` 实现（适合需要强制空行的场景）。

## 四、常用场景适配

- 笔记（Typora、Notion）：优先使用标题、列表、引用，简洁清晰即可。

- GitHub 文档：支持语法高亮、表格、链接，适合插入代码和项目说明。

- 博客/公众号：可结合图片、表格，避免过多复杂语法，提升阅读体验。

Markdown 核心是“简洁高效”，无需死记硬背，多练习几次即可熟练掌握，日常使用中可根据需求灵活组合语法，实现快速排版。
